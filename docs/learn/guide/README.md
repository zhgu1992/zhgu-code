# Guide（读者导读）

这一组文档是给“学习仓库的人”看的，不是给“直接改代码的人”看的。

目标：
1. 快速理解每个阶段到底做了什么。
2. 理解为什么要这么设计，而不是只看功能清单。
3. 建立可迁移的方法论，能带到你自己的项目里。
